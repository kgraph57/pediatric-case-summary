# Manas開発用ファイル一式

このフォルダには、Manasでアプリ開発を開始するために必要なファイルがすべて含まれています。

## 📁 ファイル構成

### 🚀 開発開始用（最重要）

1. **`Manasプロンプト.md`**
   - Manasに最初に投げるプロンプト
   - プロジェクト概要と主要機能
   - 技術スタックの提案
   - **→ これを最初にManasに投げてください**

2. **`Manas開発指示書.md`**
   - 詳細な開発仕様書
   - 機能要件、UI/UX要件、データ構造、バリデーションルールなど
   - **→ Manasに読み込ませて参照してください**

3. **`データ構造定義.json`**
   - データモデルの定義
   - Case型とSettings型の構造
   - **→ TypeScriptの型定義作成時に参照**

### 📚 参考資料

4. **`症例要約作成ルール.md`**
   - 詳細なルールと表記規則
   - バリデーションルールの詳細
   - 特殊な医学用語の表記方法

5. **`症例情報収集テンプレート.md`**
   - 入力項目の一覧
   - 各項目の説明
   - 記載順序の確認

6. **`症例要約作成ガイド.md`**
   - ユーザーの作業フロー
   - チェックリスト
   - 効率的な作業のコツ

### 📖 説明書

7. **`Manas開発用README.md`**
   - このフォルダの使い方
   - 開発時のチェックポイント
   - 重要な実装ポイント

---

## 🎯 Manasでの使い方

### Step 1: プロジェクト開始
1. `Manasプロンプト.md` の内容をコピー
2. Manasに貼り付けてプロジェクトを開始

### Step 2: 詳細仕様の読み込み
1. `Manas開発指示書.md` をManasに読み込ませる
2. 必要に応じて参照しながら開発を進める

### Step 3: データ構造の確認
1. `データ構造定義.json` を参照
2. TypeScriptの型定義を作成

### Step 4: ルールの実装
1. `症例要約作成ルール.md` を参照
2. バリデーションルールとフォーマット化ルールを実装
