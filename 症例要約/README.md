# 症例要約作成フォルダ

このフォルダには、実際に作成する症例要約を保存します。

## 📋 ファイル構成

- `症例一覧.md`: 進捗管理シート
- `症例01.md` ～ `症例30.md`: 各症例の要約ファイル

## 📝 使い方

### 方法1: 画像から情報を抽出する場合（推奨）

1. 各症例ファイルを開く（例: `症例01.md`）
2. 「📷 画像から読み取る情報」セクションを確認
3. 参照画像（`共有しない/症例要約二木良平/症例要約 二木良平jpg/症例要約 二木良平_page-0001.jpg`など）を開く
4. 画像から各項目を読み取って記載
   - 基本情報（症例番号、分野番号、患者ID、年齢、性別など）
   - 診断名
   - 主訴
   - 現病歴
   - 入院時診察所見
   - 入院時検査所見
   - 鑑別診断（必須）
   - 入院後経過
   - 家族への説明・指示（必須）
   - 退院後の経過
5. 画像から読み取った情報を基に、音声入力でテキスト化
6. 「音声入力テキスト」セクションに貼り付け
7. Webアプリ（`シンプルUI実装例.html`）で症例要約を生成
8. 生成された症例要約を「生成された症例要約」セクションにコピー
9. 確認・修正メモのチェックリストで確認
10. 必要に応じて修正

### 方法2: 直接音声入力する場合

1. 各症例ファイルを開く
2. 「音声入力テキスト」セクションに、音声入力で作成したテキストを貼り付け
3. Webアプリ（`シンプルUI実装例.html`）で症例要約を生成
4. 生成された症例要約を「生成された症例要約」セクションにコピー
5. 確認・修正メモのチェックリストで確認

## ✅ 進捗管理

各ファイルの「ステータス」を更新して進捗を管理してください：

- [ ] 未着手
- [ ] 画像から情報抽出完了
- [ ] 音声入力完了
- [ ] 症例要約生成済み
- [ ] 確認・修正済み
- [x] 完成

`症例一覧.md`でも全体の進捗を管理できます。

## 📌 注意事項

- 個人情報を含むため、このフォルダは`.gitignore`に追加されています
- 定期的にバックアップを取ることを推奨します
- 画像ファイルは「共有しない」フォルダ内にあります
