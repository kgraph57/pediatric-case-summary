# Manas用プロンプト

## プロジェクト概要

小児科専門医試験の申請に必要な30症例以上の症例要約を作成するためのWebアプリケーションを開発してください。

## 主要機能

1. **症例情報の入力と管理**：30症例以上の症例情報を入力・管理
2. **症例要約の自動生成**：入力情報からルールに基づいて症例要約を自動生成
3. **バリデーション機能**：入力値と全体の整合性をチェック
4. **チェックリスト機能**：各症例と全体のチェックリスト
5. **エクスポート機能**：PDF/Word形式でのエクスポート

## 技術スタック

- **フロントエンド**：React + TypeScript + Tailwind CSS
- **状態管理**：Zustand
- **フォーム管理**：React Hook Form + Zod
- **PDF生成**：jsPDF
- **ストレージ**：IndexedDB（ローカルストレージ）

## 詳細仕様

詳細な仕様は `Manas開発指示書.md` を参照してください。

## 実装の優先順位

1. **Phase 1（MVP）**：
   - 症例情報入力フォーム
   - 基本的なバリデーション
   - 症例要約の自動生成
   - 症例一覧表示
   - ローカルストレージ保存

2. **Phase 2**：
   - 詳細なバリデーション
   - チェックリスト機能
   - PDFエクスポート

3. **Phase 3**：
   - クラウドストレージ連携
   - ユーザーアカウント機能

## 重要なポイント

- **医療情報の取り扱い**：個人情報保護に配慮、ローカルストレージを基本とする
- **表記ルールの厳守**：数字の半角化、単位のスペース、句読点の統一など
- **30行以内の制約**：症例要約は30行以内に収める
- **必須項目のバリデーション**：鑑別診断、家族への説明・指示などは必須
