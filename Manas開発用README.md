1. `Manas開発指示書.md` をManasに読み込ませる

1. `データ構造定義.json` を参照
2. TypeScriptの型定義を作成

### Step 4: ルールの実装
1. `症例要約作成ルール.md` を参照
2. バリデーションルールとフォーマット化ルールを実装

## 🎯 重要な実装ポイント

### 1. 表記ルールの実装
- 数字の半角変換
- 単位のスペース処理（例外ルールの実装）
- 句読点の統一（、。のみ）
- 年齢表記の自動変換

### 2. バリデーションの実装
- 必須項目のチェック
- 形式チェック（数字、単位、句読点など）
- 論理チェック（外来症例数、同一疾患の重複など）
- 全体バリデーション（30例以上、各分野2例以上など）

### 3. フォーマット化の実装
- 30行以内に収める処理
- 各項目の冒頭でのみ改行
- 項目内では改行しない
- 文字サイズの設定（10.5pt～12pt）

### 4. UI/UXの実装
- シンプルで直感的なUI
- リアルタイムプレビュー
- 入力支援機能（オートコンプリート、入力例など）
- エラーメッセージの分かりやすさ

## 🎨 デザインガイドライン

### カラースキーム
- **プライマリカラー**：医療をイメージした落ち着いた色（青系）
- **アクセントカラー**：警告・注意を表す色（オレンジ・赤）
- **成功カラー**：完成・OKを表す色（緑）
- **背景色**：目に優しい色（白・ライトグレー）

### フォント
- **和文**：MS明朝、MSゴシック（または同等のフォント）
- **欧文**：Times New Roman
- **文字サイズ**：10.5pt～12pt（症例要約本文）

### レイアウト
- シンプルで直感的なUI
- 重要な情報が一目で分かるレイアウト
- モバイル対応（レスポンシブデザイン）
